Overview

All routes are served under /api/v1. There are two independent authentication systems: business users (clinic staff & patients) and platform SiteAdmins. Each uses its own JWT and Passport strategy.

AreaAuthNotes
Business (auth, users, branches)Business JWTGlobal guard; @Public() opts out
SiteAdmin + Tenant managementSiteAdmin JWTRole/permission guard
Tenant data isolation is enforced by PostgreSQL Row-Level Security on tenant_id; services also scope every query by tenant as defence in depth. All deletes are soft (deleted_at).

Setup & run

  1. Install dependencies: pnpm install
  2. Configure .env — required: DATABASE_URL, JWT_SECRET (≥ 32 chars). Optional (defaults shown): JWT_ACCESS_TTL=15m, JWT_REFRESH_TTL_DAYS=30, SITEADMIN_TOKEN_TTL=8h, BCRYPT_ROUNDS=12, PORT=3000.
  3. Generate the Prisma client: pnpm prisma generate
  4. Create the schema: pnpm prisma migrate dev --name init
  5. Enable Row-Level Security: psql "$DATABASE_URL" -f prisma/rls.sql (run the app with a least-privilege DB role — not the table owner — so RLS is enforced).
  6. Start the API: pnpm start:dev → http://localhost:3000/api/v1
Bootstrapping order: a SUPER_OWNER SiteAdmin is seeded directly in the DB (it cannot be created via the API). The SiteAdmin then creates tenants (POST /siteadmin/tenants), and each tenant's auto-created business_admin logs in to manage branches and staff.

SiteAdmin role matrix

SiteAdmin roles are cumulative — each includes every permission of the roles below it. Routes declare requirements via @RequireSiteAdminPermission(); the guard AND-checks them.

RoleAdds these permissions (on top of lower roles)
CONTENT_ADMINmaster-data:read, master-data:write
OPERATIONS_ADMINbusiness:create, business:read, business:suspend, business:reactivate, audit-logs:read, reports:operational
FULL_ADMINreports:finance
SUPER_OWNERsystem:config, subscription-plans:write, encryption:tools, siteadmin:manage
Tenant management endpoints require business:create (writes) or business:read (reads) — i.e. OPERATIONS_ADMIN and above. SiteAdmin sub-account management requires siteadmin:manage — SUPER_OWNER only.

API authorization matrix

Business routes are gated by the global JwtAuthGuard (any authenticated business user); the "intended permission" column reflects the profile permission the action maps to — used for frontend gating and ready for a future business permission guard. API Endpoints

36 endpoints across 5 areas. Auth column: public = no token, business JWT, SiteAdmin JWT.

Business Authentication

POST/api/v1/auth/login
public

Authenticate with an identifier (phone, email, or system username) + password. Returns an access + refresh token pair. tenantSlug is required for staff login.

Request body

{ "identifier": "+919876543210", "password": "MyPassword1", "tenantSlug": "city-dx" }

curl

curl -X POST https://api.kalnostics.com/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"identifier":"+919876543210","password":"MyPassword1","tenantSlug":"city-dx"}'

200 response

{ "success": true,
  "data": { "accessToken": "eyJhbGciOi…", "refreshToken": "a1b2c3…" },
  "meta": { "timestamp": "2026-06-04T10:00:00.000Z" } }
